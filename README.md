# E-Wallet - 이더리움 지갑 및 다중 서명 지갑 관리 시스템

E-Wallet은 이더리움 네트워크를 위한 개인 지갑 및 다중 서명(Multi-Signature) 지갑 관리 웹 애플리케이션입니다. React와 Ethers.js를 기반으로 구축되었으며, Sepolia 테스트넷에서 동작합니다.

## 주요 기능

### 개인 지갑 관리

- **새 지갑 생성**: BIP39/BIP44 표준을 따른 HD 지갑 생성
  - 12단어 니모닉 구문 생성
  - 메타마스크와 호환되는 경로 사용 (m/44'/60'/0'/0/0)
  
- **지갑 복구**: 여러 방법으로 지갑 복구
  - 니모닉 구문으로 복구
  - 개인키로 복구
  - 주소로 읽기 전용 열기

- **지갑 관리**
  - 여러 지갑 저장 및 관리
  - 지갑 이름 지정
  - 로컬 스토리지에 암호화 저장

### 트랜잭션 기능

- **잔액 조회**: 실시간 ETH 잔액 확인
- **송금**: 개인키로 서명하여 ETH 전송
- **트랜잭션 내역**: 트랜잭션 히스토리 조회
- **가스 설정**: 사용자 정의 가스 가격 설정

### 다중 서명 지갑 (Multi-Signature Wallet)

- **다중 서명 지갑 생성**
  - 여러 소유자 설정
  - 임계값(Threshold) 설정
  - 스마트 컨트랙트 배포

- **트랜잭션 관리**
  - 트랜잭션 제안
  - 다중 서명 승인
  - 임계값 도달 시 자동 실행
  - 서명 취소 기능

- **멤버 관리**
  - 소유자 추가 제안 및 승인
  - 소유자 제거 제안 및 승인
  - 임계값 변경 제안 및 승인

- **다중 서명 지갑 참여**
  - 주소로 기존 다중 서명 지갑 참여
  - 서명자 인증
  - 지갑 정보 조회

## 설치 및 실행

### 사전 요구사항

- Node.js 16.x 이상
- npm 또는 yarn

### 1. 저장소 클론 및 의존성 설치

```bash
# 저장소 클론 (또는 프로젝트 디렉토리로 이동)
cd e-wallet

# 의존성 설치
npm install
```

### 2. 개발 서버 실행

```bash
npm run dev
```

개발 서버가 시작되면 브라우저에서 `http://localhost:5173` (또는 표시된 포트)로 접속하세요.

### 3. 프로덕션 빌드

```bash
npm run build
```

### 4. 빌드 미리보기

```bash
npm run preview
```

## 사용 방법

### 개인 지갑 사용

#### 새 지갑 생성

1. 홈페이지에서 **"새 지갑 생성"** 클릭
2. 지갑 이름 입력
3. 생성된 **12단어 니모닉 구문**을 안전한 곳에 기록
   - 니모닉을 잃어버리면 지갑을 복구할 수 없습니다
4. 니모닉 확인 단계 완료
5. 지갑이 생성되고 대시보드로 이동

#### 지갑 복구

**니모닉으로 복구:**
1. 홈페이지에서 **"지갑 열기"** 클릭
2. **"니모닉으로 열기"** 선택
3. 지갑 이름과 12단어 니모닉 구문 입력
4. 지갑 복구 완료

**개인키로 복구:**
1. 홈페이지에서 **"지갑 열기"** 클릭
2. **"개인키로 열기"** 선택
3. 지갑 이름과 개인키 입력
4. 지갑 복구 완료

**주소로 열기 (읽기 전용):**
1. 홈페이지에서 **"지갑 열기"** 클릭
2. **"주소로 열기"** 선택
3. 지갑 주소 입력
4. 잔액 및 정보 확인 (트랜잭션 불가)

#### 송금하기

1. 지갑 대시보드에서 **"송금"** 버튼 클릭
2. 수신 주소와 금액 입력
3. 가스 가격 설정 (선택사항)
4. 트랜잭션 확인 및 전송

### 다중 서명 지갑 사용

#### 다중 서명 지갑 생성

1. 홈페이지에서 **"다중 서명 지갑"** 클릭
2. 지갑 이름 입력
3. 소유자 주소 추가 (최소 2명)
4. 임계값(Threshold) 설정
   - 임계값은 소유자 수 이하여야 함
   - 예: 소유자 3명, 임계값 2 → 2명 이상 승인 시 실행
5. 컨트랙트 배포
6. 배포 완료 후 다중 서명 지갑 대시보드로 이동

#### 다중 서명 지갑 참여

1. 홈페이지에서 **"다중 서명 지갑"** 클릭
2. **"지갑 참여"** 또는 주소로 직접 접근
3. 다중 서명 지갑 주소 입력
4. 현재 지갑이 소유자인지 확인
5. 참여 완료

#### 트랜잭션 제안 및 승인

**트랜잭션 제안:**
1. 다중 서명 지갑 대시보드에서 **"송금"** 클릭
2. 수신 주소와 금액 입력
3. 트랜잭션 제안 제출

**트랜잭션 승인:**
1. 트랜잭션 목록에서 대기 중인 트랜잭션 확인
2. **"승인"** 버튼 클릭
3. 승인 수가 임계값에 도달하면 자동 실행

**트랜잭션 실행:**
- 승인 수가 임계값에 도달하면 자동으로 실행됩니다
- 또는 수동으로 **"실행"** 버튼 클릭 가능

#### 멤버 관리

**소유자 추가:**
1. 다중 서명 지갑 설정 페이지로 이동
2. **"멤버 추가"** 클릭
3. 새 소유자 주소 입력
4. 제안 제출 후 다른 소유자들의 승인 대기

**소유자 제거:**
1. 다중 서명 지갑 설정 페이지로 이동
2. 제거할 소유자 선택
3. 제안 제출 후 다른 소유자들의 승인 대기

**임계값 변경:**
1. 다중 서명 지갑 설정 페이지로 이동
2. 새 임계값 입력
3. 제안 제출 후 다른 소유자들의 승인 대기

## 스마트 컨트랙트

### MultiSigWallet 컨트랙트

다중 서명 지갑은 Solidity로 작성된 스마트 컨트랙트입니다.

**주요 기능:**
- 여러 소유자 관리
- 임계값 기반 트랜잭션 승인
- ReentrancyGuard를 통한 재진입 공격 방지
- 관리 트랜잭션 (멤버 추가/제거, 임계값 변경)

**컨트랙트 주요 함수:**
- `submitTransaction()`: 트랜잭션 제안
- `confirmTransaction()`: 트랜잭션 승인
- `revokeConfirmation()`: 승인 취소
- `proposeAddOwner()`: 소유자 추가 제안
- `proposeRemoveOwner()`: 소유자 제거 제안
- `proposeChangeThreshold()`: 임계값 변경 제안

## 개발 가이드

### 코드 구조

- **컨텍스트**: `WalletContext.jsx`에서 전역 지갑 상태 관리
- **유틸리티**: `utils/` 디렉토리에 지갑 및 다중 서명 관련 함수
- **컴포넌트**: 재사용 가능한 UI 컴포넌트
- **페이지**: 라우트별 페이지 컴포넌트

### 주요 유틸리티 함수

**wallet.js:**
- `createNewWallet()`: 새 HD 지갑 생성
- `recoverWalletFromMnemonic()`: 니모닉으로 복구
- `recoverWalletFromPrivateKey()`: 개인키로 복구
- `saveWalletData()` / `loadWalletData()`: 로컬 스토리지 관리

**multisig.js:**
- `deployMultiSigWallet()`: 다중 서명 지갑 배포
- `proposeTransaction()`: 트랜잭션 제안
- `confirmTransaction()`: 트랜잭션 승인
- `getMultiSigTransactions()`: 트랜잭션 목록 조회

### 네트워크 설정

현재 Sepolia 테스트넷을 사용합니다.

### 스마트 컨트랙트 컴파일

스마트 컨트랙트는 `contractCompiler.js`를 통해 컴파일되며, 바이트코드는 `bytecode.js`에 저장됩니다.
